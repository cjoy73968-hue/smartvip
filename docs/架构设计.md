## 二、技术选型理由

### 后端：Java 17 + Spring Boot 3

| 选择 | 理由 |
|------|------|
| Java 17 | 长期支持版本，性能强，生态成熟，适合复杂业务逻辑 |
| Spring Boot 3 | 最新稳定版，对 Java 17 支持好，自动配置简化开发 |
| MyBatis-Plus | 半自动化 ORM，轻量可控，对复杂 SQL 友好，避免黑盒 |
| Redis | Session 共享 + 定时任务分布式锁 + 热数据缓存 |

**为什么不选 Spring Cloud？**
项目规模不需要微服务拆分，单体足够支撑所有功能，微服务会增加运维复杂度。

### 前端：Vue3 + TypeScript + Element Plus

| 选择 | 理由 |
|------|------|
| Vue3 | 组合式 API 更灵活，适合中后台管理界面 |
| TypeScript | 类型检查减少运行时错误，代码可维护性强 |
| Element Plus | 组件丰富文档完善，适合快速开发管理后台 |

**为什么不选 React？**
Vue 上手快，Element Plus 是成熟的中后台 UI 库，开发效率更高。

### 小程序：微信原生

| 选择 | 理由 |
|------|------|
| 原生小程序 | 微信生态最深度集成，授权/支付/地图等 API 最完善 |
| 原生 + JavaScript | 比 uni-app 少一层转译，性能更好 |

## 三、核心业务模型

```
用户（users）
    ├── 充值余额 / 返利余额（双子账户）
    ├── 推荐关系（referrer_id → 推荐人）
    ├── 会员有效期（membership_start / membership_expire）
    └── 地址（addresses）

订单（consumption_orders）
    └── 关联推荐人，产生返利记录（rebate_logs）

礼品派送（gift_delivery_orders）
    ├── 周期派送（差价模式）→ 本周期不可取消
    └── 一次性派送（全额模式）→ 付款后不可取消

商户配置（merchant_config）
    ├── 返利比例（默认5%）
    ├── 会员价格/有效期
    ├── 配送城市/派送周期
    └── 默认礼品
```

## 四、定时任务设计

| 任务 | 频率 | 说明 |
|------|------|------|
| 会员有效期检查 | 每日凌晨 | 检查到期会员，关闭权益（派送/返利） |
| 会员到期提醒 | 每日凌晨 | 提前30天发送微信提醒 |
| 自动订阅扣款 | 每日凌晨 | 检查即将到期且开启自动订阅的会员 |
| 礼品派送触发 | 按配置的派送时间 | 按商家配置的周期自动生成派送订单 |
| 返利到账处理 | 每日凌晨 | 确认收货7天后的订单，发放返利 |
| 数据报表生成 | 每月1日 | 统计上月数据 |

**防重复执行**：使用 Redis 分布式锁，确保集群环境下任务不重复。

## 五、数据库设计要点

### 事务保证

- 余额操作（充值/消费/退款）：使用事务 + 乐观锁（version 字段）
- 返利计算：幂等性设计，防止重复发放

### 索引设计

```sql
users: phone（登录查询）, referral_code（推荐码）, referrer_id（推荐关系）, membership_expire（有效期筛选）
consumption_orders: user_id, referrer_id（返利统计）, rebate_status（到账处理）
gift_delivery_orders: order_type + status（派送管理）, user_id
rebate_logs: referrer_id, status（到账/追回）
```

### 逻辑删除

所有数据表使用 `isDeleted` 字段做逻辑删除，数据不物理删除，方便对账和追溯。

## 六、微信集成

### 登录授权

```
微信授权 → 获取 openid → 首次注册（绑定手机号）→ 正式会员
```

### 微信支付

```
充值：用户支付 → 微信回调 → 增加充值余额
消费：余额扣款 → 无需微信支付（余额内部流转）
提现：用户申请 → 商家打款 → 手动确认（不走微信提现接口）
```

### 模板消息

| 场景 | 模板类型 | 触发时机 |
|------|---------|---------|
| 新派送订单 | 通知商家 | 定时任务生成派送订单 |
| 余额不足 | 通知用户 | 派送时余额不足 |
| 返利到账 | 通知用户 | 确认收货7天后 |
| 会员到期提醒 | 通知用户 | 到期前30天 |

## 七、部署架构

```
腾讯云轻量应用服务器（2核4G）
    │
    ├── Docker Compose
    │   ├── Java 后端服务（端口8080）
    │   ├── MySQL 8.0（端口3306）
    │   └── Redis 7（端口6379）
    │
    └── Nginx（反向代理）
        ├── /api/* → 后端服务
        └── /admin/* → Vue3 静态资源（前端构建产物）
```

## 八、安全设计

1. **密码加密**：商家后台使用 BCrypt 加密存储
2. **敏感配置**：所有密钥放在 application.yml，不提交到 Git
3. **防SQL注入**：MyBatis-Plus 参数化查询，禁用字符串拼接
4. **CSRF/XSS**：Spring Security 配置Headers
5. **接口鉴权**：商家后台使用 Session + 过滤器验证登录状态
